# Декомпозиция проекта: Telegram-бот ↔ CRM

**Проект:** Автоматизация обработки входящих обращений, фиксация коммуникации, управление клиентской базой
**Стек:** Python 3.12 · aiogram 3.x · FastAPI · PostgreSQL · Redis + arq · Docker · amoCRM (REST + Chat API)

---

## Как пользоваться этим документом

- Каждая задача рассчитана на **один заход в Claude Code (~20–40 мин)**: понятный вход, конкретный артефакт на выходе, критерий готовности.
- Идти **сверху вниз по фазам** — задачи внутри фазы зависят от предыдущих.
- Формулировку задачи можно копировать в Claude Code почти как есть, добавив контекст репозитория.
- `[ ]` — чекбоксы для отметки выполнения.
- Фазы помечены **(условная)** включаются только если соответствующее требование подтверждено клиентом.

---

## Предусловия (решить ДО старта разработки)

Без этих ответов часть задач заблокирована. Это блокеры из ранее согласованного чек-листа:

- [ ] **CRM подтверждена** (amoCRM / Bitrix24) + тариф + доступ к API.
- [ ] **Решение build-vs-buy**: пишем кастом (этот план) или берём готовый коннектор.
- [ ] **Телефон**: обязателен или опционален (влияет на фазу 2 — запрос контакта).
- [ ] **Групповые чаты**: нужны или нет (включает условную фазу 2.7 и модель `group_chats`).
- [ ] **Тяжёлые файлы (>20/50 МБ)**: нужны или нет (включает условную фазу 7 — local Bot API).
- [ ] **Инфраструктура**: чей сервер, чей домен/SSL, чей аккаунт BotFather.

## Порядок и зависимости

```
Фаза 0 → Фаза 1 → Фаза 2 ─┐
                          ├→ Фаза 3 → Фаза 4 → Фаза 5 → Фаза 6 → Фаза 8
        (групп. чаты 2.8) ┘
                          (тяжёлые файлы → Фаза 7, параллельно после Ф.2)
```

- Фазы 0–1 — фундамент, строго первыми.
- Фазы 2 и 3 можно частично вести параллельно (разные люди).
- Фаза 5 зависит от 4 (нужны созданные контакты/сделки).
- Фазы 6 и 8 — сквозные, добивать ближе к концу.
